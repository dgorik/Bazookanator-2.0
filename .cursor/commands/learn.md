# Learn Command

Analyze and explain the current file in depth.

## Instructions

When the user runs this command, provide a comprehensive explanation of the code in the currently focused file:

1. **Overview**: What is this file's purpose? What problem does it solve?

2. **Key Components**: Break down the main functions, components, types, or classes. Explain what each does.

3. **Data Flow**: How does data move through this file? What are the inputs and outputs?

4. **Dependencies**: What external libraries or internal modules does this file rely on? Why?

5. **Patterns Used**: Identify any design patterns, React patterns (hooks, memoization), or architectural patterns.

6. **Gotchas**: Point out any non-obvious behavior, edge cases, or common pitfalls.

7. **How It Connects**: Explain how this file fits into the larger codebase. What calls it? What does it call?

## Output Format

Use clear headings and code snippets where helpful. Keep explanations concise but thorough. Use bullet points for lists. Highlight important concepts in **bold**.
